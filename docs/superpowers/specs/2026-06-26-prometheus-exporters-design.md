# Prometheus Exporters: MySQL, HDFS, Trino

**Date:** 2026-06-26
**Status:** Approved

## Overview

Add Prometheus metrics coverage for three workloads that currently have none:
- **MySQL** (`mysqld_exporter`) â€” InnoDB cluster in `mysql` ns
- **HDFS** (`jmx_exporter` Java agent on namenode) â€” raw StatefulSet in `hdfs` ns
- **Trino** (`jmx_exporter` Java agent on coordinator + workers) â€” Helm chart in `trino` ns

All three feed into the existing Alloy â†’ Mimir pipeline.

## Component 1: MySQL (`mysqld_exporter`)

### Vault secret

Add to `scripts/vault-secrets.template.yaml`:

```yaml
- path: mysql/monitoring
  fields: [username, password]
  values: { username: monitoring, password: "" }
```

`username` is fixed (`monitoring`); empty `password` is auto-generated by `seed-vault.sh`. Run `scripts/seed-vault.sh` after adding.

### New Helm workload: `gitops/workloads/helm/mysqld-exporter/`

**`config.json`**
```json
{
  "chart_repo": "https://prometheus-community.github.io/helm-charts",
  "chart_name": "prometheus-mysql-exporter",
  "chart_version": "2.x.x"
}
```
Pin to the latest stable release of `prometheus-mysql-exporter` at implementation time.

**`values.yaml`**
- `mysql.host`: `mysql-cluster.mysql.svc`
- `mysql.port`: `3306`
- Auth from ExternalSecret-managed secret `mysql-monitoring-credentials`
- Service port `9104` (mysqld_exporter default)

**`extras/00-root-credentials.yaml`** â€” ExternalSecret pulling `mysql/root` from `vault-homelab` into secret `mysql-root` in the `mysqld-exporter` namespace (the bootstrap Job needs root auth but cannot access the secret in the `mysql` namespace).

**`extras/01-bootstrap-job.yaml`** â€” Argo CD Sync hook (`argocd.argoproj.io/hook: Sync`, `BeforeHookCreation`), sync-wave `-1`:
- Connects to `mysql-cluster.mysql.svc:3306` using `mysql-root` secret (from `extras/00-root-credentials.yaml`)
- Creates user `monitoring@'%'` if not exists
- Grants: `PROCESS, REPLICATION CLIENT, SELECT ON performance_schema.*`

**`extras/02-monitoring-credentials.yaml`** â€” ExternalSecret pulling `mysql/monitoring` from `vault-homelab` ClusterSecretStore into secret `mysql-monitoring-credentials`.

### Alloy scrape block

```
discovery.kubernetes "mysqld_exporter" { role = "endpoints", namespaces = ["mysqld-exporter"] }
discovery.relabel "mysqld_exporter" { keep port 9104 }
prometheus.scrape "mysql" { job_name = "mysql" }
```

## Component 2: HDFS (`jmx_exporter` agent on namenode)

### Changes to `gitops/workloads/raw/hdfs/manifests/hdfs.yaml`

**New ConfigMap `hdfs-jmx-config`** in `hdfs` ns:
```yaml
startDelaySeconds: 0
ssl: false
rules:
  - pattern: "Hadoop<service=NameNode, name=(.+)><>(.+)"
    name: "hadoop_namenode_$1_$2"
    labels:
      service: namenode
  - pattern: ".*"
```

**Namenode StatefulSet additions:**
- Init container `jmx-exporter-init`: image `bitnami/jmx-exporter:0.20.0`, copies `/opt/bitnami/jmx-exporter/jmx_prometheus_javaagent.jar` â†’ shared `emptyDir` at `/jmx`
- Volume `jmx-agent`: `emptyDir: {}`
- Volume `jmx-config`: ConfigMap `hdfs-jmx-config`, key `config.yaml`
- Namenode container: mount `jmx-agent` at `/jmx/agent`, mount `jmx-config` at `/jmx/config.yaml` (subPath)
- Env `HADOOP_NAMENODE_OPTS`: `-javaagent:/jmx/agent/jmx_prometheus_javaagent.jar=9087:/jmx/config.yaml`
- Container port `9087` (name: `jmx`)

**Namenode Service:** add port `jmx: 9087 â†’ 9087`.

Datanode is not instrumented in this iteration (namenode metrics cover capacity, block counts, live nodes).

### Alloy scrape block

```
discovery.kubernetes "hdfs" { role = "endpoints", namespaces = ["hdfs"] }
discovery.relabel "hdfs" { keep port 9087, keep label app.kubernetes.io/name=namenode }
prometheus.scrape "hdfs" { job_name = "hdfs" }
```

## Component 3: Trino (`jmx_exporter` agent on coordinator + workers)

### New extra: `gitops/workloads/helm/trino/extras/jmx-config.yaml`

ConfigMap `trino-jmx-config` in `trino` ns:
```yaml
startDelaySeconds: 0
ssl: false
rules:
  - pattern: "trino.execution<name=QueryManager><>(.+)"
    name: "trino_query_manager_$1"
  - pattern: "trino.execution<name=TaskManager><>(.+)"
    name: "trino_task_manager_$1"
  - pattern: "trino.memory<name=ClusterMemoryManager><>(.+)"
    name: "trino_cluster_memory_$1"
  - pattern: "trino.failuredetector<name=HeartbeatFailureDetector><>(.+)"
    name: "trino_failure_detector_$1"
  - pattern: ".*"
```

### Changes to `gitops/workloads/helm/trino/values.yaml`

For both **coordinator** and **worker** roles, add:

```yaml
extraVolumes:
  - name: jmx-agent
    emptyDir: {}
  - name: jmx-config
    configMap:
      name: trino-jmx-config

extraVolumeMounts:
  - name: jmx-agent
    mountPath: /jmx/agent
  - name: jmx-config
    mountPath: /jmx/config.yaml
    subPath: config.yaml

initContainers:
  - name: jmx-exporter-init
    image: bitnami/jmx-exporter:0.20.0
    command: ["cp", "/opt/bitnami/jmx-exporter/jmx_prometheus_javaagent.jar", "/jmx/agent/"]
    volumeMounts:
      - name: jmx-agent
        mountPath: /jmx/agent

additionalJVMConfig:
  - "-javaagent:/jmx/agent/jmx_prometheus_javaagent.jar=9087:/jmx/config.yaml"

additionalExposedPorts:
  jmx:
    internalPort: 9087
    port: 9087
    protocol: TCP
```

### Alloy scrape block

```
discovery.kubernetes "trino" { role = "endpoints", namespaces = ["trino"] }
discovery.relabel "trino" { keep port 9087 }
prometheus.scrape "trino" { job_name = "trino" }
```
